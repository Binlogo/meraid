## Problem Statement

meraid is a Rust-based terminal Mermaid diagram renderer. The goal is to establish open-source influence by positioning it as an AI-friendly CLI tool for AI coding agents (Claude Code, Codex, etc.).

## What Makes This Cool

- Pure Rust implementation with zero external dependencies
- 6 built-in themes (default, terra, neon, mono, amber, phosphor)
- Pipeline-friendly with stdin support
- **The killer feature**: Designed specifically for AI agents to use — clean CLI interface, parseable output, AI-friendly error messages

## Target Users & Use Cases

1. **Primary**: Claude Code / Codex users who want to render Mermaid diagrams in terminal
2. **Secondary**: Developers who use Mermaid in documentation
3. **Tertiary**: Technical presenters/bloggers who need quick terminal-based diagram rendering

## Current Status

- Already implements: Flowchart, Sequence Diagram, State Diagram, Pie Chart, Class Diagram
- 4 commits on main branch
- Basic theme system in place

## Constraints

- Pure Rust (no external dependencies)
- Terminal-focused (ASCII-first rendering)
- Phase 1: Focus on most common diagram types

## Premises

1. meraid is a **general-purpose CLI** that happens to be AI-friendly — NOT an AI-only tool
2. Phased approach: start with most common chart types, expand later
3. The open-source goal is to build influence in the AI tooling community

## Approaches Considered

### Approach A: Minimal Viable
- Focus on core 4 diagram types
- Quick crates.io release
- Risk: Low, but differentiation is weak

### Approach B: AI-First (CHOSEN)
- Enhance with AI-friendly CLI interface
- Add JSON output mode for programmatic use
- Create OpenClaw/Claude Code Skill integration
- Optimize error messages for AI parsing

### Approach C: Complete Ecosystem
- Web preview, VS Code plugin, GitHub Action
- Risk: Over-engineering, long timeline

## Recommended Approach

**Approach B: AI-First** — Balance quick delivery with clear differentiation.
